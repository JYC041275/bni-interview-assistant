# 🌐 GitHub Pages 自訂網域設置教學

## 📋 前置準備

您需要:
- ✅ 一個已註冊的網域名稱(例如: `example.com`)
- ✅ 網域註冊商的 DNS 管理權限
- ✅ GitHub 倉庫的管理權限

---

## 🎯 設置步驟

### 步驟 1: 決定使用的網域類型

您可以選擇以下任一種:

#### 選項 A: 使用子網域(推薦)
- 例如: `bni.yourdomain.com`
- 優點: 設置簡單,不影響主網域
- 適合: 將此應用作為網站的一部分

#### 選項 B: 使用主網域
- 例如: `yourdomain.com`
- 優點: 網址簡潔
- 適合: 整個網域專門用於此應用

#### 選項 C: 使用 www 子網域
- 例如: `www.yourdomain.com`
- 優點: 傳統且專業
- 適合: 作為主要網站使用

---

### 步驟 2: 在 GitHub 設置自訂網域

#### 2.1 前往 GitHub 倉庫設置

1. 打開瀏覽器,前往:
   ```
   https://github.com/JYC041275/bni-interview-assistant/settings/pages
   ```

2. 登入您的 GitHub 帳號

#### 2.2 添加自訂網域

1. 找到 **"Custom domain"** 區塊
2. 在輸入框中輸入您的網域,例如:
   - `bni.yourdomain.com` (子網域)
   - `yourdomain.com` (主網域)
   - `www.yourdomain.com` (www 子網域)
3. 點擊 **"Save"**

#### 2.3 等待 DNS 檢查

- GitHub 會開始檢查 DNS 設置
- 此時會顯示警告訊息(這是正常的)
- 我們接下來會設置 DNS

---

### 步驟 3: 設置 DNS 記錄

根據您選擇的網域類型,設置不同的 DNS 記錄:

#### 選項 A: 子網域設置(例如 `bni.yourdomain.com`)

在您的 DNS 管理面板中添加以下記錄:

| 類型  | 名稱/主機 | 值/目標                                    | TTL  |
|-------|-----------|-------------------------------------------|------|
| CNAME | bni       | jyc041275.github.io                       | 3600 |

**說明**:
- **類型**: 選擇 `CNAME`
- **名稱**: 輸入子網域名稱(例如 `bni`)
- **值**: 輸入 `jyc041275.github.io`(您的 GitHub Pages 網址)
- **TTL**: 通常設置為 3600(1小時)

#### 選項 B: 主網域設置(例如 `yourdomain.com`)

在您的 DNS 管理面板中添加以下 **4 個 A 記錄**:

| 類型 | 名稱/主機 | 值/目標        | TTL  |
|------|-----------|---------------|------|
| A    | @         | 185.199.108.153 | 3600 |
| A    | @         | 185.199.109.153 | 3600 |
| A    | @         | 185.199.110.153 | 3600 |
| A    | @         | 185.199.111.153 | 3600 |

**說明**:
- **類型**: 選擇 `A`
- **名稱**: 輸入 `@`(代表根網域)
- **值**: 依序輸入上面 4 個 IP 地址
- **TTL**: 通常設置為 3600

#### 選項 C: www 子網域設置

添加以下 2 個記錄:

| 類型  | 名稱/主機 | 值/目標                | TTL  |
|-------|-----------|----------------------|------|
| CNAME | www       | jyc041275.github.io  | 3600 |
| A     | @         | 185.199.108.153      | 3600 |

(如果要完整支持,建議添加全部 4 個 A 記錄)

---

### 步驟 4: 常見 DNS 管理面板設置示例

#### GoDaddy
1. 登入 GoDaddy
2. 前往「我的產品」→「DNS」
3. 點擊「新增」
4. 選擇記錄類型並填入資訊
5. 點擊「儲存」

#### Cloudflare
1. 登入 Cloudflare
2. 選擇您的網域
3. 前往「DNS」→「記錄」
4. 點擊「新增記錄」
5. 填入資訊並儲存

#### Namecheap
1. 登入 Namecheap
2. 前往「Domain List」→「Manage」
3. 選擇「Advanced DNS」
4. 點擊「Add New Record」
5. 填入資訊並儲存

#### 其他註冊商
- 尋找「DNS 管理」、「DNS 設置」或「域名解析」
- 添加 CNAME 或 A 記錄
- 按照上面的表格填入資訊

---

### 步驟 5: 等待 DNS 生效

#### 時間
- DNS 更改通常需要 **5-30 分鐘**生效
- 某些情況下可能需要 **24-48 小時**

#### 檢查 DNS 是否生效

**方式 A: 使用線上工具**
1. 前往: https://dnschecker.org/
2. 輸入您的網域
3. 選擇記錄類型(CNAME 或 A)
4. 查看全球各地的 DNS 狀態

**方式 B: 使用終端機命令**

檢查 CNAME 記錄:
```bash
dig bni.yourdomain.com CNAME +short
```
應該顯示: `jyc041275.github.io`

檢查 A 記錄:
```bash
dig yourdomain.com A +short
```
應該顯示 GitHub 的 IP 地址

---

### 步驟 6: 啟用 HTTPS(強烈建議)

DNS 生效後:

1. 回到 GitHub Pages 設置頁面:
   ```
   https://github.com/JYC041275/bni-interview-assistant/settings/pages
   ```

2. 找到 **"Enforce HTTPS"** 選項

3. 勾選 **"Enforce HTTPS"**

4. GitHub 會自動申請並配置 SSL 證書(Let's Encrypt)

5. 等待幾分鐘,HTTPS 就會啟用

---

### 步驟 7: 更新 Vite 配置(重要!)

因為您使用了自訂網域,需要修改 `vite.config.ts`:

#### 7.1 修改 base 路徑

打開 `vite.config.ts`,將:
```typescript
base: '/bni-interview-assistant/',
```

改為:
```typescript
base: '/',
```

#### 7.2 重新部署

```bash
git add vite.config.ts
git commit -m "Update base path for custom domain"
git push
npm run deploy
```

## 🔧 常見問題

### Q1: DNS 已設置但網站無法訪問
**A**: 
- 等待更長時間(最多 48 小時)
- 清除瀏覽器快取
- 使用無痕模式測試
- 檢查 DNS 記錄是否正確

### Q2: 顯示 "404 - There isn't a GitHub Pages site here"
**A**:
- 確認 GitHub Pages 設置中的自訂網域正確
- 確認 DNS 記錄指向正確的 GitHub Pages 網址
- 重新部署: `npm run deploy`

### Q3: HTTPS 無法啟用
**A**:
- 確保 DNS 已完全生效
- 等待 24 小時後再試
- 取消勾選 HTTPS,等待幾分鐘後再勾選

### Q4: 網站樣式跑掉了
**A**:
- 確認已修改 `vite.config.ts` 的 `base` 為 `'/'`
- 重新構建並部署

### Q5: 想要同時支持 www 和非 www
**A**:
- 設置兩個網域的 DNS 記錄
- 在 GitHub Pages 設置中選擇主要網域
- GitHub 會自動重定向
